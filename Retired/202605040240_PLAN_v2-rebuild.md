## Objective

Rebuild Diarizer v2 against the **faster-whisper + pyannote 3.3.2** stack. Reuse the stack-agnostic v2 modules (config, output, CLI scaffold, 24 of 33 tests). Rewrite the pipeline and preprocessing modules. Add cheap, fail-fast validation gates between stages. Verify the dependency triple installs cleanly **before** any code commits — this is the discipline that the abandoned WhisperX PLAN lacked.

## Context — what changed

The WhisperX-anchored PLAN 202605040220 was paused 2026-05-04 because (a) install on Windows + CUDA hit a torch/ctranslate2/pyannote pin conflict, (b) WhisperX is "three models in a trench coat" so the architectural simplification was illusory. RESEARCH 202605040230 surveyed the SOTA and recommends building directly on faster-whisper (the same model WhisperX wraps) and pyannote 3.3.x (the same diariser WhisperX wraps), without the WhisperX coordination layer.

Concrete audio measurements (RESEARCH 202605040220) of three operator-supplied recordings (live meeting, denoised remote, Teams) show the input envelope is wide: from ideally-prepped 16 kHz mono with 86 % voiced content (Teams) to spectrally-damaged 48 kHz stereo with 75 % silence (denoised remote). This rules out a fixed filter chain and motivates **adaptive preprocessing with measurement gates**.

### Step 4 — Rewrite v2/preprocessing.py with measurement + adaptive filters

New module surface:
```python
@dataclass
class AudioMeasurement:
    duration_s: float
    sample_rate: int
    channels: int
    rms_dbfs: float
    peak_dbfs: float
    noise_floor_dbfs: float
    voiced_fraction: float
    spectral_centroid_hz: float

def measure(input_path: Path) -> AudioMeasurement: ...
def preprocess(input_path: Path, output_path: Path,
               config: PreprocessingConfig,
               measurement: AudioMeasurement | None = None) -> Path: ...
```

`measure()` is the substrate for Gates 0 and 1 (see Step 6). `preprocess()` consumes the measurement and decides:
- Always: decode to PCM, resample to 16 kHz mono if input ≠ that.
- If `RMS dBFS < -32`: apply `loudnorm=I=-23:LRA=7:TP=-2`.
- If `noise_floor_dbfs > -40 AND estimated SNR < 20 dB AND config.denoise`: apply `afftdn=nf=-25`. Default-OFF.
- If `spectral_centroid_hz > 6000`: log warning (already-processed input).

Verify-after-preprocess: re-measure output; if post-RMS < pre-RMS by >6 dB, abort and retry without filters.

**verify:** `python -c "from v2.preprocessing import measure, preprocess"` succeeds.

**acceptance:** Round-trip on all three recordings produces 16 kHz mono PCM output with measurable post-state.

### Step 5 — Rewrite v2/pipeline.py against faster-whisper + pyannote

New `Pipeline` class:

```python
class Pipeline:
    def __init__(self, config: Config): ...
    def transcribe(self, audio_path: Path) -> TranscriptionResult: ...
```

Internally:
1. `measurement = preprocessing.measure(audio_path)` → Gate 0.
2. `processed_path = preprocessing.preprocess(audio_path, ..., measurement)` → Gate 1.
3. Coarse Silero VAD pass (CPU, fast) → Gate 2.
4. `pyannote.audio.Pipeline.from_pretrained("pyannote/speaker-diarization-3.1")` → diarise → Gate 3 (warn) → unload + clear cache.
5. `faster_whisper.WhisperModel(name, device, compute_type=int8_float16)` with `vad_filter=True`, `beam_size=1`, `word_timestamps=True`, `condition_on_previous_text=False` → transcribe → Gate 4 (warn) → unload + clear cache.
6. Attribution: per ASR segment, take diariser speaker with max overlap → "Unknown Speaker" on tie / no overlap → Gate 5 (warn).
7. Return `TranscriptionResult` (existing dataclass).

CUDA OOM handling: catch `torch.cuda.OutOfMemoryError`, retry once with `compute_type="int8"` if was `int8_float16`. Log clearly. If still OOMs, raise with a "lower the model size in config" message.

**verify:** `python -c "from v2.pipeline import Pipeline"` succeeds; type-check by running existing config tests against the updated Pipeline imports.

### Step 6 — Implement validation gates as a separate module

New `v2/gates.py`:
- `ingest_gate(measurement) -> GateResult` — Gate 0.
- `preprocess_damage_gate(pre, post) -> GateResult` — Gate 1.
- `voice_presence_gate(voiced_fraction) -> GateResult` — Gate 2.
- `diarisation_sanity_gate(speaker_count, share_distribution) -> GateResult` — Gate 3.
- `asr_sanity_gate(transcript, voiced_duration) -> GateResult` — Gate 4.
- `output_sanity_gate(result, output_path) -> GateResult` — Gate 5.

`GateResult`: `{passed: bool, severity: BLOCK|WARN, message: str, metrics: dict}`. Gates 0-2 escalate to BLOCK on failure (abort run). Gates 3-5 escalate to WARN (log, continue).

Pipeline.transcribe() owns gate dispatch; gates are pure functions, easily unit-testable.

**verify:** `python -c "from v2.gates import ingest_gate, voice_presence_gate"` succeeds.

**Step 4 — preprocessing.py rewrite: DONE 2026-05-04.**

New surface:
- `AudioMeasurement` dataclass (frozen) — duration, SR, channels, RMS dBFS, peak dBFS, noise/speech floor dBFS, voiced fraction, spectral centroid Hz, derived `estimated_snr_db` property.
- `measure(path) -> AudioMeasurement` — librosa-backed, lazy import. Cost ~hundreds of ms per minute of audio.
- `preprocess(input, output, config, measurement=None) -> Path` — adaptive FFmpeg invocation:
  - Always: decode + 16 kHz mono.
  - Loudnorm only when measurement is None or `rms_dbfs < -32`.
  - Denoise (afftdn) only when explicitly enabled in config AND `noise_floor > -40 dBFS` AND `SNR < 20 dB`. Default off.
  - Spectral-centroid > 6 kHz → log warning (audio likely pre-processed upstream).
- Back-compat alias `preprocess_audio = preprocess` so existing test_preprocessing.py tests pass unchanged.

**Step 5 — pipeline.py rewrite: DONE 2026-05-04.**

Replaces WhisperX with direct calls:
- `from faster_whisper import WhisperModel` (lazy, inside `_load_whisper()`)
- `from pyannote.audio import Pipeline as PyannotePipeline` (lazy, inside `_run_diarisation()`)

Stage flow (matches RESEARCH 230 design):
1. Ingest: `measure(audio_path)` → `ingest_gate` (BLOCK) → raise `PipelineGateFailure` on fail.
2. Preprocess (if enabled): `preprocess(...)` to tempfile → re-measure → `preprocess_damage_gate` (BLOCK).
3. Voice presence: `voice_presence_gate(voiced_fraction)` (BLOCK on near-silence).
4. Diarisation (if enabled, HF token resolved): pyannote 3.1 pipeline → speaker timeline → `diarisation_sanity_gate` (WARN).
5. ASR: `WhisperModel.transcribe(beam_size=1, vad_filter=True, word_timestamps=True, condition_on_previous_text=False)` → `asr_sanity_gate` (WARN).
6. Attribution: per-segment max-overlap with diariser timeline → "Unknown Speaker" on no overlap.

OOM handling: `_load_whisper()` retries with `compute_type="int8"` if first attempt raises an OOM-shaped error.

Memory hygiene: each backend loaded inside its stage function, deleted at end, `torch.cuda.empty_cache()` + `torch.cuda.ipc_collect()` in `finally`.

`TranscriptionResult` extended with `speaker_count`, `gate_results` (list of dicts), `measurement` (dict). Existing TXT/JSON/SRT writers continue to work because they consume `segments` only.

**Step 6 — gates.py: DONE 2026-05-04.**

Six gates as pure functions, two severities (BLOCK / WARN, plus INFO for passing-with-detail):
- `ingest_gate(measurement, *, min_duration_s=10, valid_sample_rates=..., max_channels=8, min_rms_dbfs=-50)` — BLOCK.
- `preprocess_damage_gate(pre, post, *, rms_drop_threshold_db=6, centroid_drop_threshold_hz=1000)` — BLOCK.
