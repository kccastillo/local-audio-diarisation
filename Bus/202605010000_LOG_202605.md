## Lessons Learned

Captured during the plan-pipeline build (PLAN 202605011400 + children, 2026-05-01). Architectural, process, and user-preference observations worth preserving.

**Architectural:** Mechanical vs conceptual review are different operations needing different model tiers — `audit-sufficiency` (opus) then `audit-haiku-safe` (sonnet) both required before advancing. Subagents do NOT inherit parent skill registry; each agent declares `skills: [...]` in frontmatter. Wire format (`<pipeline-result>` JSON) is for immediate parse; PLAN frontmatter fields are durable disk record for re-entry idempotency. `verify:` checks pieces-in-place; `acceptance:` exercises spec delivery (decision 25). Triage decisions into Already-locked / Mechanically-forced / Real-judgement-call before surfacing to Human (decision 15). Subagent-driven exception handling beats orchestrator supervision (Erlang "let it crash").

**Process:** Focused agent probes catch fatal assumptions early (probe-1 caught subagent skill registry isolation + undocumented streaming before code). PLAN-as-design-spec emerged: Opus authors design PLANs directly; Verification is the contract. Re-running checks after each design change kept blocker-count visible and forced honest tallies.

**User preferences (Ken):** Options-with-recommendation always; push back on overengineering; Ken's metaphors are architectural directives; token economy > feature breadth; codified behaviour > memory crutches; verify spec, not just steps; fast feedback loops.

**Conventions:** Skills declare contracts in XML blocks; subagents preload skills; outcome enum is `success | revision_needed | exception`; orchestrator dispatches by `(phase, outcome)` tuple; audit loops use per-stage iteration counters (MAX=5); wire format is `<pipeline-result>` JSON block.

**Closeout F/14:** Plan-pipeline orchestrator + supporting skill ecosystem (ideate, audit-sufficiency, audit-haiku-safe, plan-pipeline) built end-to-end. F1 finding (subagent permission context, GH #37730) fixed via Option C (plan-executor variants use filesystem/Read/Edit/Write + `python -c` instead of raw Bash). Smoke-test PLAN 2100 validated F1 fix.
