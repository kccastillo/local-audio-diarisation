## Objective

Build note-jot, a small utility skill for jotting notes during sessions. The skill exists primarily as a dogfood target for the freshly-built plan-pipeline orchestrator — it is the first non-bootstrap PLAN to flow through the system end-to-end (parent: 202605011440 dogfood spec).

## Context

**Why this skill exists.** note-jot is not a tool with a real day-to-day use case. It exists because the plan-pipeline orchestrator (built in parent PLAN 202605011400, child PLANs 1410-1430) needs a real artefact to validate against — something concrete enough to exercise every phase (drafting → drafted → checked → executing → outcome-verifying → complete → retired) without being so trivial that the audit phase has nothing to chew on.

**Design bar.** Small but real. The Survey-phase decisions (file location, format, invocation contract) should be defensible but lightweight — pick something credible, not perfect. Retention can be short; PLANs typically execute same-day or within the same week, so anything jotted has a short useful life regardless.

**Out of scope (anti-scope) — confirmed at Clarify close:**
- Long-term searchability across weeks/months.
- Cross-project usage (project-local is fine).
- Polish (rich formatting, colour, tagging, schema validation).

**Implicit constraints — confirmed at Clarify close:**
- Invokable via `Skill("note-jot", ...)` like other skills.
- Produces a real, observable artefact when called (a timestamped line in a file) so the dogfood's `acceptance:` checks have something to verify.
- No setup required — invoke and it works (no config files, no path arguments needed for first call).

## Steps

1. Create the skill directory and SKILL.md.
   - Create directory `.claude/skills/note-jot/` if it does not exist.
   - Create file `.claude/skills/note-jot/SKILL.md` with this exact content:
     ```
     ---
     name: note-jot
     description: Jots a one-line timestamped note to a daily log file under `notes/YYYY-MM-DD.md` in the project. Single positional argument is the note text. Trigger phrases: "jot a note", "note this", "log a note", "note-jot".
     ---
     
     **Procedure:** See [workflows/jot-procedure.md](workflows/jot-procedure.md).
     
     <essential_principles>
     Single-purpose: append one timestamped line per invocation; do nothing else.
     File path is project-local: `notes/YYYY-MM-DD.md` relative to the current working directory.
     If the day-file does not exist, create it with a single H1 header `# Notes — YYYY-MM-DD` before appending.
     Append-only — never modify existing lines.
     </essential_principles>
     
     <inputs>
     - `note_text: string` — single positional argument; the note's text content.
     </inputs>
     
     <output_schema>
     - Side effect: one line appended to `notes/YYYY-MM-DD.md`.
     - Return: confirmation string with the file path and the appended line.
     </output_schema>
     
     <constraints>
     - Never write to any path outside `notes/`.
     - Never overwrite existing notes.
     - Never require config files or path arguments.
     </constraints>
     
     <success_criteria>
     - `notes/YYYY-MM-DD.md` exists after invocation.
     - The expected line `- HH:MM:SS — <note text>` is present at end of file.
     - If the day-file was newly created, it begins with the H1 header.
     </success_criteria>
     ```

2. Create the workflow file describing the jot procedure.
   - Create directory `.claude/skills/note-jot/workflows/` if it does not exist.
   - Create file `.claude/skills/note-jot/workflows/jot-procedure.md` with this exact content:
     ```
     # Note-jot procedure
     
     On each invocation:
     
     1. Read the positional argument as the note text.
     2. Compute today's date as `YYYY-MM-DD` (use `date +%Y-%m-%d` or equivalent).
     3. Compute current time as `HH:MM:SS` (use `date +%H:%M:%S` or equivalent).
     4. Determine the target file path: `notes/YYYY-MM-DD.md` (relative to the current working directory).
     5. If the target file does not exist:
        - Ensure the `notes/` directory exists (`mkdir -p notes`).
        - Create the file with the line `# Notes — YYYY-MM-DD` followed by a blank line.
     6. Append the line `- HH:MM:SS — <note text>` to the file.
     7. Return a confirmation string: `Jotted to notes/YYYY-MM-DD.md: - HH:MM:SS — <note text>`.
     
     Done. The skill performs no other action.
     ```

3. Add `notes/` to `.gitignore`.
   - Open the repo's `.gitignore` (at the repo root).
   - If a line `notes/` is not already present, append it on its own line.
   - If a `notes/` line already exists, leave the file untouched.

4. Smoke-test the procedure manually.
   - Run the procedure as a shell oneliner to confirm behaviour:
     `mkdir -p notes && echo "- $(date +%H:%M:%S) — smoke test from execute-plan" >> notes/$(date +%Y-%m-%d).md`
   - Confirm the file `notes/$(date +%Y-%m-%d).md` exists and contains the smoke-test line.
